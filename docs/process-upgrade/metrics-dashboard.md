# 过程度量看板定义与复盘机制

## 1. 看板目标

- 形成“效率-质量-AI效能”三维度的统一度量体系。
- 支撑迭代复盘与流程策略调整（模板、门禁、评审规则）。

## 2. 指标分层

## 2.1 交付效率指标

| 指标 | 定义 | 采集方式 | 频率 |
|---|---|---|---|
| Lead Time | 需求进入开发到上线完成时长 | 需求系统 + CI/CD记录 | 周 |
| PR Cycle Time | PR创建到合并时长 | 代码托管平台 | 周 |
| Review Turnaround | 提交后首次有效评审耗时 | PR事件流 | 周 |

## 2.2 质量稳定性指标

| 指标 | 定义 | 采集方式 | 频率 |
|---|---|---|---|
| Defect Density | 千行代码缺陷数 | 缺陷系统 + 变更规模 | 周 |
| Defect Escape Rate | 线上缺陷占总缺陷比例 | 线上告警/缺陷系统 | 周 |
| Regression Failure Rate | 回归执行失败占比 | 测试平台 | 周 |
| Incident Rate | 线上事故率 | 监控与值班系统 | 周 |

## 2.3 测试有效性指标

| 指标 | 定义 | 采集方式 | 频率 |
|---|---|---|---|
| Automation Coverage | 自动化覆盖率 | 测试报告 | 周 |
| Case Effectiveness | 有效检出缺陷的用例占比 | 测试管理平台 | 双周 |
| MTTR-TestFailure | 测试失败定位平均时长 | CI日志 + 工单 | 周 |

## 2.4 AI效能指标

| 指标 | 定义 | 采集方式 | 频率 |
|---|---|---|---|
| AI Suggestion Adoption | AI建议采纳率 | PR评论/记录 | 周 |
| First Pass Yield | 首次通过门禁比例 | CI门禁结果 | 周 |
| Rework Ratio | 返工任务占比 | 需求状态流转 | 双周 |

## 3. 指标阈值建议（起步版）

- Defect Escape Rate：不高于历史基线。
- First Pass Yield：持续提升，且不以牺牲质量为代价。
- Sev1/Sev2：目标为0，出现即触发专项复盘。

## 4. 看板使用机制

## 4.1 例行节奏

- 每周：研发/测试例会查看本周波动指标。
- 每双周：进行迭代复盘，分析根因与流程改进动作。
- 每月：评估门禁阈值、模板质量与AI提示词资产。

## 4.2 异常触发机制

满足任一条件触发专项分析：

- 缺陷逃逸率连续两期上升；
- 回归失败率显著上升；
- 首次通过率明显下降；
- 高频出现同类问题（同模块、同类型缺陷）。

## 5. 复盘输出模板（建议）

每次复盘至少输出以下内容：

1. 指标波动事实（数据+趋势）。
2. 根因分析（流程/设计/实现/测试/工具）。
3. 改进动作（负责人、验证方式、截止条件）。
4. 下周期跟踪指标与目标。
